# Умные Шторы - Smart Curtain Control

Приложение для управления умными шторами через NodeMCU32 с использованием Python сервера.

## 🎉 Версия 2.0 - Все настройки в приложении!

**Теперь не нужно менять код!** Все параметры (IP адреса, порты, интервалы обновления) настраиваются прямо из мобильного приложения через удобный экран настроек.

### ✨ Новые возможности:
- ⚙️ Полноценный экран настроек
- 🔌 Проверка соединения с сервера
- 🔄 Сброс к настройкам по умолчанию
- 📤 Экспорт настроек
- 📳 Вибрация при нажатии (настраивается)
- 🎚️ Настраиваемый интервал автообновления

**📖 Быстрый старт:** См. файл `БЫСТРЫЙ_СТАРТ.md`
**⚙️ Подробно о настройках:** См. файл `НАСТРОЙКИ.md`

## 🚀 Установка и запуск

### 1. Python сервер

```bash
cd server
pip install -r requirements.txt
python curtain_server.py
```

Сервер запустится на `http://0.0.0.0:5000`

### 2. NodeMCU32 (ESP32)

1. Откройте `server/nodemcu32_example.ino` в Arduino IDE
2. Установите библиотеки:
   - WiFi (встроенная)
   - WebServer (встроенная)
   - ArduinoJson
3. Настройте WiFi credentials:
   ```cpp
   const char* ssid = "YOUR_WIFI_SSID";
   const char* password = "YOUR_WIFI_PASSWORD";
   ```
4. Укажите IP адрес Python сервера:
   ```cpp
   const char* serverIP = "192.168.1.100";
   ```
5. Загрузите код на ESP32

### 3. Android приложение

1. Откройте проект в Android Studio
2. Синхронизируйте Gradle
3. Запустите приложение на устройстве или эмуляторе
4. Нажмите кнопку ⚙️ (настройки) и укажите:
   - URL сервера: `http://192.168.1.100:5000` (ваш IP)
   - Название штор: `Моя штора`
   - ID штор: `curtain_1`
   - IP NodeMCU: `192.168.1.101` (IP вашего ESP32)
   - Порт NodeMCU: `80`
5. Нажмите "Проверить соединение"
6. Нажмите "Сохранить настройки"

## 🔌 API эндпоинты

### Python сервер

| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| GET | `/api/ping` | Проверка доступности сервера |
| GET | `/api/curtains` | Список всех штор |
| GET | `/api/curtain/state/<id>` | Состояние конкретной шторы |
| POST | `/api/curtain/command` | Отправка команды управления |
| POST | `/api/nodemcu/status` | Обновление статуса от NodeMCU |
| GET/POST | `/api/config` | Управление конфигурацией |

### Формат команды (JSON):

```json
{
  "curtain_id": "curtain_1",
  "action": "open|close|stop|set_position",
  "position": 50,
  "timestamp": 1234567890
}
```

### Формат ответа (JSON):

```json
{
  "success": true,
  "message": "Команда выполнена",
  "curtain_state": {
    "id": "curtain_1",
    "name": "Штора 1",
    "position": 50,
    "is_moving": false,
    "status": "stopped",
    "last_update": 1234567890
  }
}
```

## ⚙️ Конфигурация

### Android приложение
- **Все настройки** доступны через кнопку ⚙️ в приложении
- Настройки хранятся в `SharedPreferences`
- **9 настраиваемых параметров:**
  1. URL Python сервера
  2. Название штор
  3. ID штор
  4. IP адрес NodeMCU
  5. Порт NodeMCU
  6. Интервал автообновления
  7. Таймаут соединения
  8. Вибрация (вкл/выкл)
  9. Звук (вкл/выкл)

### Python сервер
- Редактируйте переменные в начале `curtain_server.py`:
  ```python
  NODEMCU_IP = "192.168.1.101"
  NODEMCU_PORT = 80
  ```

### NodeMCU32
- WiFi credentials в коде Arduino
- Пины мотора настраиваются в константах:
  ```cpp
  const int MOTOR_PIN1 = 12;
  const int MOTOR_PIN2 = 14;
  const int MOTOR_ENABLE = 27;
  ```

## 🔧 Подключение мотора к ESP32

```
ESP32          L298N Motor Driver
GPIO 12   -->  IN1
GPIO 14   -->  IN2
GPIO 27   -->  ENA (PWM)
GND       -->  GND
5V        -->  +5V

Motor Driver   DC Motor
OUT1      -->  Motor +
OUT2      -->  Motor -
```

## 📝 Требования

## 🐛 Отладка

### Проблемы с подключением:
1. Проверьте, что сервер запущен: `curl http://localhost:5000/api/ping`
2. Проверьте firewall на компьютере с сервером
3. Убедитесь, что Android приложение имеет разрешение INTERNET
4. Проверьте, что `usesCleartextTraffic="true"` в AndroidManifest.xml

### Логи:
- **Android**: Logcat в Android Studio
- **Python**: Консоль сервера
- **ESP32**: Serial Monitor (115200 baud)
