# 📱 Руководство по настройкам приложения "Умные Шторы"

## ✨ Что нового?

Теперь **ВСЕ настройки** можно изменять прямо из мобильного приложения! Больше не нужно лезть в код!

---

## 🎯 Как открыть настройки?

1. Запустите приложение
2. Нажмите на кнопку **⚙️** (шестеренка) в правом нижнем углу
3. Откроется полноценный экран настроек

---

## 🔧 Доступные настройки

### 🌐 Подключение к серверу

#### **URL Python сервера**
- **Что это:** Адрес вашего компьютера с Python сервером
- **Формат:** `http://IP_АДРЕС:5000`
- **Пример:** `http://192.168.1.100:5000`
- **Как узнать IP:**
  - Windows: откройте CMD и введите `ipconfig`
  - Найдите "IPv4 Address" в разделе вашей WiFi сети

#### **Таймаут соединения (мс)**
- **Что это:** Время ожидания ответа от сервера
- **По умолчанию:** 5000 мс (5 секунд)
- **Рекомендуется:** 3000-10000 мс
- **Зачем менять:** Если сервер медленный, увеличьте значение

---

### 🪟 Настройки штор

#### **Название штор**
- **Что это:** Отображаемое имя в приложении
- **По умолчанию:** "Штора 1"
- **Примеры:** "Гостиная", "Спальня", "Балкон"
- **Зачем менять:** Для удобства, если у вас несколько штор

#### **ID штор**
- **Что это:** Уникальный идентификатор для сервера
- **По умолчанию:** `curtain_1`
- **Формат:** Латинские буквы и цифры, без пробелов
- **Примеры:** `curtain_1`, `living_room`, `bedroom_main`
- **Важно:** Должен совпадать с ID на сервере!

---

### 📡 Настройки NodeMCU (ESP32)

#### **IP адрес NodeMCU**
- **Что это:** IP адрес вашего ESP32 в локальной сети
- **По умолчанию:** `192.168.1.101`
- **Как узнать:** Смотрите в Serial Monitor при загрузке ESP32
- **Пример:** `192.168.1.101`

#### **Порт NodeMCU**
- **Что это:** Порт веб-сервера на ESP32
- **По умолчанию:** `80`
- **Обычно:** Не нужно менять
- **Диапазон:** 1-65535

---

### ⚡ Поведение приложения

#### **Интервал автообновления (мс)**
- **Что это:** Как часто приложение запрашивает состояние штор
- **По умолчанию:** 5000 мс (5 секунд)
- **Рекомендуется:** 3000-10000 мс
- **Зачем менять:**
  - Меньше значение = быстрее обновление, но больше нагрузка
  - Больше значение = экономия батареи и трафика

#### **Вибрация при нажатии**
- **Что это:** Вибрация при нажатии на кнопки
- **По умолчанию:** Включено ✅
- **Зачем отключать:** Если вибрация раздражает

#### **Звуковые уведомления**
- **Что это:** Звуковые сигналы при действиях
- **По умолчанию:** Выключено ❌
- **Зачем включать:** Для дополнительной обратной связи

---

## 🎮 Кнопки действий

### 💾 Сохранить настройки
- Сохраняет все изменения
- Проверяет корректность введенных данных
- Автоматически возвращает на главный экран

### 🔌 Проверить соединение
- Проверяет доступность сервера
- Показывает результат проверки
- Не сохраняет настройки (можно тестировать)

### 🔄 Сбросить к значениям по умолчанию
- Восстанавливает все настройки по умолчанию
- Требует подтверждения
- Полезно, если что-то сломалось

### 📤 Экспорт настроек
- Показывает все текущие настройки
- Копирует их в буфер обмена
- Полезно для отладки или резервного копирования

2. Найдите строку с IP адресом ESP32
3. Запомните его, например: `192.168.1.101`

1. Откройте приложение
2. Нажмите ⚙️
3. Введите:
   - **URL сервера:** `http://192.168.1.100:5000`
   - **Название штор:** `Моя штора`
   - **ID штор:** `curtain_1`
   - **IP NodeMCU:** `192.168.1.101`
   - **Порт NodeMCU:** `80`
4. Нажмите **"Проверить соединение"**
5. Если всё ОК, нажмите **"Сохранить настройки"**

## ⚠️ Частые ошибки и решения

### ❌ "Сервер не отвечает"
**Причины:**
- Python сервер не запущен
- Неправильный IP адрес
- Устройства в разных WiFi сетях
- Firewall блокирует соединение

**Решение:**
1. Проверьте, что сервер запущен
2. Проверьте IP адрес командой `ipconfig`
3. Убедитесь, что телефон и компьютер в одной WiFi сети
4. Временно отключите firewall для теста

### ❌ "Ошибка: проверьте числовые значения"
**Причина:** Введены буквы вместо цифр

**Решение:**
- Порт: только цифры (например: `80`)
- Интервал: только цифры (например: `5000`)
- Таймаут: только цифры (например: `5000`)

### ❌ "URL должен начинаться с http://"
**Причина:** Неправильный формат URL

**Решение:**
- Правильно: `http://192.168.1.100:5000`
- Неправильно: `192.168.1.100:5000`

### ❌ "Порт должен быть от 1 до 65535"
**Причина:** Введен неправильный порт

**Решение:**
- Обычно используется порт `80`
- Для Python сервера: `5000`

## 💡 Советы по оптимизации

### Для экономии батареи:
- Увеличьте интервал автообновления до 10000 мс
- Отключите вибрацию
- Отключите звук

### Для быстрой работы:
- Уменьшите интервал автообновления до 3000 мс
- Уменьшите таймаут до 3000 мс

### Для стабильности:
- Используйте стандартные значения
- Не ставьте слишком маленький таймаут
- Не ставьте слишком частое обновление
